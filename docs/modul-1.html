Modul 1: Persediaan Laravel & Asas Kerangka

Hasil Pembelajaran

Pelajar dapat menyediakan persekitaran pembangunan Laravel sepenuhnya serta memahami struktur asas Model-View-Controller (MVC), penggunaan routing, serta Blade templating dalam pembangunan antaramuka sistem web.

Bab 1.1: Pemasangan Perisian Asas

Sebelum memulakan pembangunan sistem menggunakan Laravel, pelajar perlu menyediakan beberapa perisian asas yang akan membentuk persekitaran pembangunan. Langkah-langkah pemasangan adalah seperti berikut:

Langkah 1: Pemasangan 7-Zip

  1. Layari laman web rasmi: https://www.7-zip.org/download.html.
  2. Pilih versi yang bersesuaian dengan sistem pengoperasian.
  3. Muat turun dan jalankan pemasangan dengan tetapan lalai tanpa sebarang pengubahsuaian.

Langkah 2: Pemasangan Git

  1. Layari laman web: https://git-scm.com/downloads.
  2. Pilih dan muat turun versi mengikut sistem pengoperasian.
  3. Jalankan pemasangan secara lalai.

Langkah 3: Pemasangan XAMPP

  1. Layari: https://www.apachefriends.org/download.html.
  2. Pilih versi PHP 8.2.x.
  3. Jalankan pemasangan dengan konfigurasi lalai.

Langkah 4: Pemasangan Node.js

  1. Lawati: https://nodejs.org/en/download.
  2. Pilih versi LTS (contoh: 22.x.x seperti yang dinyatakan).
  3. Jalankan pemasangan tanpa sebarang perubahan pada tetapan.

Langkah 5: Pemasangan Composer

  1. Akses: https://getcomposer.org/download/.
  2. Pilih versi stabil (contoh: 2.8.x) dan pastikan untuk menambah PHP ke PATH anda semasa pemasangan jika pilihan tersebut ada.
  3. Selepas pemasangan selesai, buka terminal dan jalankan perintah berikut untuk konfigurasi repositori Packagist dan tetapan IPv6 (jika perlu):
composer config --global repo.packagist composer https://packagist.org
netsh interface ipv6 set prefixpolicy ::ffff:0:0/96 55 4

Disarankan untuk menambah direktori projek ke dalam pengecualian Windows Defender:

  • Windows Security > Virus & threat protection > Manage settings > Add or remove exclusions
  • Tambah direktori seperti C:\xampp\htdocs

Langkah 6: Pemasangan Visual Studio Code

Visual Studio Code (VS Code) merupakan editor kod sumber ringan yang sangat popular dalam pembangunan web dan Laravel.

Langkah pemasangan:
  1. Layari: https://code.visualstudio.com/Download.
  2. Pilih versi yang sesuai (Windows/macOS/Linux).
  3. Jalankan pemasangan dengan tetapan lalai.
  4. Buka VS Code dan aktifkan "Auto Save": File > Auto Save.

Bab 1.2: Penciptaan Projek Laravel Menggunakan Composer

  1. Jalankan perintah berikut di terminal untuk navigasi ke direktori kerja anda (contoh: htdocs XAMPP):
    cd C:\xampp\htdocs
  2. Cipta projek Laravel baru (gantikan nama_projek dengan nama projek anda):
    composer create-project laravel/laravel:~10.0 nama_projek --prefer-dist
  3. Masuk ke direktori projek dan buka dalam VS Code:
    cd nama_projek
    code .
  4. Klik "Yes, I trust the authors" apabila diminta oleh VS Code.
  5. Klik ikon + di terminal VS Code untuk membuka terminal baharu dalam direktori projek.

Bab 1.3: Konfigurasi Fail .env dan Pelaksanaan Migrasi Awal

  1. Buka fail .env di root projek anda dan ubah konfigurasi pangkalan data seperti berikut (sesuaikan jika perlu):
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=unicms # Nama pangkalan data anda
DB_USERNAME=root   # Nama pengguna pangkalan data anda
DB_PASSWORD=      # Kata laluan pangkalan data anda (kosongkan jika tiada)
  1. Cipta pangkalan data unicms (atau nama yang anda tetapkan) di phpMyAdmin atau alat pengurusan pangkalan data pilihan anda. Pastikan set aksara adalah utf8mb4_unicode_ci.
  2. Dalam terminal VS Code (di dalam direktori projek), jalankan migrasi untuk mencipta jadual-jadual asas Laravel:
    php artisan migrate
  3. Mulakan pelayan pembangunan Laravel:
    php artisan serve
  4. Buka pelayar web dan layari alamat yang dipaparkan (biasanya http://127.0.0.1:8000) untuk melihat laman utama Laravel.

Bab 1.4: Pengenalan kepada Konsep Model-View-Controller (MVC) dalam Laravel

Konsep MVC adalah corak seni bina perisian yang memisahkan aplikasi kepada tiga komponen utama yang saling berkaitan:

  • Model: Bertanggungjawab untuk mengurus data aplikasi. Ia berinteraksi dengan pangkalan data (contoh: jadual pengguna, produk), melakukan operasi CRUD (Create, Read, Update, Delete), dan melaksanakan logik perniagaan yang berkaitan dengan data. Dalam Laravel, Model biasanya terletak di direktori app/Models.
  • View: Bertanggungjawab untuk memaparkan data kepada pengguna. Ia adalah antaramuka pengguna (UI) aplikasi yang dilihat oleh pengguna dalam pelayar. Dalam Laravel, Views biasanya ditulis menggunakan enjin templat Blade (fail .blade.php) dan terletak di direktori resources/views.
  • Controller: Bertindak sebagai perantara antara Model dan View. Ia menerima input daripada pengguna (melalui request HTTP dari route), memproses input tersebut (mungkin dengan memanggil kaedah pada Model untuk mendapatkan atau mengubah data), dan kemudian memilih View yang sesuai serta menghantar data yang diperlukan ke View tersebut untuk dipaparkan. Controller dalam Laravel terletak di direktori app/Http/Controllers.

Bab 1.5: Penggunaan Routing Asas melalui Fail web.php

Fail routes/web.php adalah tempat anda mentakrifkan kebanyakan route untuk aplikasi web anda. Route ini akan diakses melalui pelayar dan memetakan URL ke tindakan Controller atau fungsi tertentu.

Contoh route ke halaman utama yang memaparkan view welcome.blade.php:

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('welcome');
});

Contoh route yang memanggil kaedah index dalam HomeController apabila URL /home diakses:

// Pastikan anda telah mencipta HomeController: php artisan make:controller HomeController
use App\Http\Controllers\HomeController;

Route::get('/home', [HomeController::class, 'index']);

Contoh route dengan parameter dinamik (ID pelajar) yang memanggil kaedah show dalam PelajarController:

// Pastikan anda telah mencipta PelajarController: php artisan make:controller PelajarController
use App\Http\Controllers\PelajarController;

Route::get('/pelajar/{id}', [PelajarController::class, 'show']);
// Dalam PelajarController, kaedah show akan menerima $id
// public function show(string $id) { /* ... */ }

Bab 1.6: Penggunaan Blade Templating dan Penyediaan Layout Asas

Blade adalah enjin templat yang mudah tetapi berkuasa yang disediakan bersama Laravel. Ia tidak menghalang anda daripada menggunakan kod PHP biasa dalam view anda dan menyediakan ciri-ciri seperti pewarisan templat, bahagian (sections), komponen, dan arahan kawalan yang ringkas.

Contoh layout utama (resources/views/layouts/app.blade.php):

<!DOCTYPE html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>@yield('title', config('app.name', 'Laravel')) - {{ config('app.name', 'Laravel') }}</title>
    {{-- Pautan ke CSS global --}}
    <link rel="stylesheet" href="{{ asset('css/app.css') }}"> {{-- Contoh jika menggunakan Vite/Mix --}}
    @stack('styles') {{-- Untuk CSS khusus halaman --}}
</head>
<body class="font-sans antialiased">
    <div class="min-h-screen bg-gray-100">
        <header class="bg-white shadow">
            {{-- Masukkan navigasi utama di sini, contoh: @include('partials.navbar') --}}
            <div class="max-w-7xl mx-auto py-6 px-4 sm:px-6 lg:px-8">
                 <h1 class="text-3xl font-bold text-gray-900">@yield('header_title', 'Dashboard')</h1>
            </div>
        </header>

        <!-- Page Content -->
        <main>
            <div class="py-12">
                <div class="max-w-7xl mx-auto sm:px-6 lg:px-8">
                    <div class="bg-white overflow-hidden shadow-sm sm:rounded-lg">
                        <div class="p-6 bg-white border-b border-gray-200">
                            @yield('content')
                        </div>
                    </div>
                </div>
            </div>
        </main>

        <footer class="bg-white border-t border-gray-200">
            <div class="max-w-7xl mx-auto py-4 px-4 sm:px-6 lg:px-8 text-center text-gray-500">
                &copy; {{ date('Y') }} {{ config('app.name', 'Laravel') }}. Hak Cipta Terpelihara.
                {{-- Contoh: @include('partials.footer-content') --}}
            </div>
        </footer>
    </div>
    {{-- Pautan ke JS global --}}
    <script src="{{ asset('js/app.js') }}" defer></script> {{-- Contoh jika menggunakan Vite/Mix --}}
    @stack('scripts') {{-- Untuk JavaScript khusus halaman --}}
</body>
</html>

Bab 1.7: Pemasangan Laravel Bootstrap Auth Scaffolding dan Spatie Laravel Permission

Arahan Pemasangan Laravel UI (Bootstrap Auth Scaffolding):

composer require laravel/ui
php artisan ui bootstrap --auth
npm install
npm run build # atau npm run dev untuk pembangunan

Nota: Untuk projek Laravel yang lebih baru (Laravel 9+), Laravel Breeze atau Laravel Jetstream adalah pilihan scaffolding pengesahan yang lebih moden dan disyorkan. Jika anda menggunakan Vite (lalai dalam Laravel 9+), pastikan konfigurasi Vite anda betul.

Arahan Pemasangan Spatie Laravel Permission:

composer require spatie/laravel-permission
php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"
php artisan migrate

Konfigurasi Model Pengguna (app/Models/User.php):

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;
use Spatie\Permission\Traits\HasRoles; // Import trait HasRoles

class User extends Authenticatable
{
    use HasFactory, Notifiable, HasRoles; // Guna trait HasRoles

    /**
     * The attributes that are mass assignable.
     *
     * @var array<int, string>
     */
    protected $fillable = [
        'name',
        'email',
        'password',
    ];

    /**
     * The attributes that should be hidden for serialization.
     *
     * @var array<int, string>
     */
    protected $hidden = [
        'password',
        'remember_token',
    ];

    /**
     * Get the attributes that should be cast.
     *
     * @return array<string, string>
     */
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password' => 'hashed',
        ];
    }
}

Daftar Middleware dalam app/Http/Kernel.php (bahagian $middlewareAliases):

// Dalam fail app/Http/Kernel.php
// ...
protected $middlewareAliases = [ // atau $routeMiddleware untuk versi Laravel yang lebih lama
    // ... middleware lain ...
    'role' => \Spatie\Permission\Middleware\RoleMiddleware::class,
    'permission' => \Spatie\Permission\Middleware\PermissionMiddleware::class,
    'role_or_permission' => \Spatie\Permission\Middleware\RoleOrPermissionMiddleware::class,
];
// ...

Nota: Untuk Laravel 11+, fail Kernel.php telah diubahsuai. Middleware biasanya didaftarkan dalam fail bootstrap/app.php menggunakan kaedah ->withMiddleware(). Sila rujuk dokumentasi Spatie Laravel Permission dan Laravel untuk cara pendaftaran middleware yang terkini mengikut versi Laravel anda.

Cipta Seeder untuk Roles, Permissions, dan Users:

php artisan make:seeder RolePermissionSeeder
php artisan make:seeder UserSeeder

Isi kandungan database/seeders/RolePermissionSeeder.php:

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use Spatie\Permission\Models\Role;
use Spatie\Permission\Models\Permission;
use Spatie\Permission\PermissionRegistrar;

class RolePermissionSeeder extends Seeder
{
    public function run(): void
    {
        // Reset cached roles and permissions
        app()[PermissionRegistrar::class]->forgetCachedPermissions();

        // Cipta permissions asas (sesuaikan mengikut keperluan aplikasi)
        Permission::updateOrCreate(['name' => 'view users', 'guard_name' => 'web']);
        Permission::updateOrCreate(['name' => 'create users', 'guard_name' => 'web']);
        Permission::updateOrCreate(['name' => 'edit users', 'guard_name' => 'web']);
        Permission::updateOrCreate(['name' => 'delete users', 'guard_name' => 'web']);

        Permission::updateOrCreate(['name' => 'view posts', 'guard_name' => 'web']);
        Permission::updateOrCreate(['name' => 'create posts', 'guard_name' => 'web']);

        // Cipta roles
        $adminRole = Role::updateOrCreate(['name' => 'admin', 'guard_name' => 'web']);
        $staffRole = Role::updateOrCreate(['name' => 'staff', 'guard_name' => 'web']);
        $studentRole = Role::updateOrCreate(['name' => 'student', 'guard_name' => 'web']);

        // Berikan semua permission kepada admin
        $adminRole->givePermissionTo(Permission::all());

        // Berikan permission tertentu kepada staff
        $staffRole->givePermissionTo(['view posts', 'create posts']);

        // Berikan permission tertentu kepada student
        $studentRole->givePermissionTo(['view posts']);
    }
}

Isi kandungan database/seeders/UserSeeder.php:

namespace Database\Seeders;

use Illuminate\Database\Seeder;
use App\Models\User;
use Illuminate\Support\Facades\Hash;

class UserSeeder extends Seeder
{
    public function run(): void
    {
        $admin = User::updateOrCreate(
            ['email' => 'admin@example.com'],
            ['name' => 'Admin User', 'password' => Hash::make('password')]
        );
        $admin->assignRole('admin');

        $staff = User::updateOrCreate(
            ['email' => 'staff@example.com'],
            ['name' => 'Staff User', 'password' => Hash::make('password')]
        );
        $staff->assignRole('staff');

        $student = User::updateOrCreate(
            ['email' => 'student@example.com'],
            ['name' => 'Student User', 'password' => Hash::make('password')]
        );
        $student->assignRole('student');
    }
}

    public function run(): void

Bab 1.8: Penyediaan Struktur Blade Menggunakan Templat AdminLTE

Dalam pembangunan antaramuka sistem web yang konsisten dan profesional, penggunaan templat siap guna seperti AdminLTE adalah sangat digalakkan. Laravel menyediakan kemudahan untuk memisahkan komponen antaramuka kepada fail-fail Blade yang modular dan boleh digunakan semula, seperti layout.app, sidebar, header, footer, dan halaman home.

Langkah 1: Muat Turun dan Salin Fail Templat AdminLTE

  1. Layari https://adminlte.io dan muat turun versi terkini AdminLTE (contohnya AdminLTE 3.x).
  2. Ekstrak fail yang dimuat turun.
  3. Salin folder dist dan plugins dari AdminLTE yang diekstrak ke dalam folder public/ projek Laravel anda.
  4. Anda juga mungkin memerlukan beberapa fail CSS atau JS tambahan dari AdminLTE atau untuk ikon (seperti Font Awesome yang biasanya disertakan atau disyorkan oleh AdminLTE). Letakkan ini juga dalam folder public/css atau public/js mengikut kesesuaian.

Pastikan struktur folder anda di dalam public/ lebih kurang seperti berikut (nama folder mungkin sedikit berbeza bergantung pada versi AdminLTE):

public/ ├── css/ (Untuk CSS tambahan anda atau AdminLTE) ├── js/ (Untuk JS tambahan anda atau AdminLTE) ├── dist/ (Dari AdminLTE - mengandungi css, js, img) └── plugins/ (Dari AdminLTE - mengandungi pelbagai plugin) └── ... (Fail lain dalam public seperti index.php, .htaccess)

Langkah 5: Konfigurasi Laluan (Route)

Pastikan anda mempunyai route untuk halaman utama yang memerlukan pengesahan. Dalam fail routes/web.php, anda mungkin sudah ada atau boleh tambah/ubah suai route berikut:

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\HomeController; // Jika anda guna controller

// Laluan untuk halaman utama selepas log masuk
Route::get('/home', [HomeController::class, 'index'])
    ->middleware(['auth']) // Pastikan pengguna telah log masuk
    ->name('home'); // Beri nama pada route

// Jika belum ada HomeController, anda boleh cipta dengan: php artisan make:controller HomeController
// Dan dalam HomeController, kaedah index boleh memaparkan view home:
// public function index() { return view('home'); }

// Atau, jika anda tidak menggunakan controller untuk route mudah ini:
// Route::get('/home', function () {
//     return view('home');
// })->middleware(['auth'])->name('home');

// Pastikan juga route pengesahan asas dari Laravel UI atau Breeze/Jetstream berfungsi
// Auth::routes(); // Untuk Laravel UI

Jika anda menggunakan Laravel Breeze atau Jetstream, route /dashboard mungkin merupakan route lalai selepas log masuk, bukan /home. Sesuaikan mengikut scaffolding pengesahan yang anda gunakan.
